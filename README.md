# Шафраноф — веб-приложение ресторана (Чита)

## Готово: все 10 модулей — полный стек от Hero до email-уведомлений

### Email-уведомления
- `shared/lib/mailer.ts` — nodemailer, fail-soft (нет SMTP в `.env` — письма
  просто не шлются, ничего не падает; то же правило, что и у Redis-кэша)
- При новом заказе / брони / банкетной заявке письмо уходит на
  `RESTAURANT_NOTIFY_EMAIL` — админ узнаёт о заявке, не заходя в панель
- Банкетная заявка дополнительно шлёт гостю письмо-подтверждение, если он
  указал email (единственная форма, где email не обязателен, но собирается)
- Заказы и брони собирают только телефон (не email), поэтому подтверждение
  гостю по ним — по логике продукта, звонком/смс, не email


### Запуск (первый раз)
```bash
cp .env.example .env          # переменные окружения (не забудьте JWT_SECRET!)
npm install                   # поставит зависимости + сгенерирует Prisma Client
npm run docker:up             # поднимет Postgres, Redis, Adminer
npm run db:migrate            # создаст таблицы
npm run db:seed               # заполнит меню/галерею/отзывы + создаст админа
npm run dev
```
Сид выведет в консоль логин/пароль дефолтного админа
(по умолчанию `admin@shafranoff.ru` / `shafranoff2026`).

- Публичный сайт — http://localhost:3000
- Админка — http://localhost:3000/admin (редирект на `/admin/login`, если нет сессии)
- Adminer (GUI для БД) — http://localhost:8080

### Запуск (повторный)
```bash
npm run docker:up
npm run dev
```

### Admin panel — что теперь можно делать
- **Заказы** (`/admin/orders`) — менять статус (Новый → Подтверждён → Готовится
  → В доставке → Завершён/Отменён) прямо из таблицы
- **Меню** (`/admin/menu`) — добавлять блюда (форма), удалять, редактировать
  цену инлайн (клик по цене → ввод → Enter/blur), переключать доступность
  ("86" блюда одним кликом, не трогая карточку в публичном меню)
- **Брони** (`/admin/bookings`) — отменять бронь с подтверждением
- **Банкеты** (`/admin/banquets`) — менять статус заявки (Новая → Связались →
  Подтверждена/Отклонена)
- **Аналитика** (`/admin/analytics`) — выручка за 7 дней, средний чек, топ-5 блюд
- Все мутации защищены дважды: middleware на Edge (401 для `/api/admin/*`,
  редирект для страниц) + повторная проверка сессии внутри каждого route handler
- Изменение/добавление/удаление блюда сбрасывает Redis-кэш меню
  (`invalidateCache('menu:v1')`) — публичное меню видит изменения сразу,
  а не только через 5 минут TTL
- Удаление блюда, которое уже есть в чьём-то заказе, заблокировано на уровне
  БД (FK-ограничение) — API возвращает понятную ошибку и предлагает вместо
  этого отметить блюдо недоступным

### Полный список реализованного
1. **Hero + главная** — видео-hero, «нить шафрана» как сигнатурный элемент, JSON-LD
2. **Меню** — категории, фильтры, индикатор остроты
3. **Корзина + заказ** — Zustand, `/checkout`, серверный пересчёт цены
4. **Бронирование столов** — живые слоты по вместимости зала
5. **Банкетные заявки** — лид-форма с типом мероприятия и бюджетом
6. **Галерея + отзывы** — masonry, лайтбокс, рейтинги
7. **Docker + Prisma + Postgres + Redis** — вся запись/чтение через реальную БД
8. **Auth (JWT)** — httpOnly cookie, bcrypt, rate limit на логин
9. **Admin panel** — CRUD меню, смена статусов заказов/броней/банкетов, аналитика
10. **Email-уведомления** — админ узнаёт о новой активности сразу

### Что нужно проверить/добавить перед деплоем
- Видео/фото в `public/` (см. README внутри `public/images/` и `public/videos/`)
- Реальные адрес/телефон/координаты в `app/layout.tsx` (JSON-LD)
- Обязательно сменить `JWT_SECRET` и пароль дефолтного админа в production
- Реальные ссылки на отзовики в `features/reviews`
